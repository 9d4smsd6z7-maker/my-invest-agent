# Thread 模板

> Agent 使用说明：Thread 适合深度分析、投研拆解、复盘总结。参考 data/raw/tweets/ 中的历史 Thread 风格，结合以下结构生成草稿，保存到 output/content/。
> 建议每条推文 100-200 字，总长度 5-12 条。

---

## 结构框架

### 第 1 条 — 钩子（Hook）

**目标**：让读者忍不住看下去

```
[一个让人意外或好奇的结论/问题]

[为什么这件事值得花 X 分钟]

🧵 完整分析 ↓
```

---

### 第 2-3 条 — 背景铺垫

**目标**：建立共同认知，定义问题

```
[背景信息，确保读者明白在讨论什么]
[数据/事实支撑]
```

---

### 第 4-7 条 — 核心论点

**目标**：逐点展开，每条一个论点

```
[论点 N]：

• [支撑数据/事实]
• [逻辑推导]
• [实例或对比]
```

---

### 第 8-9 条 — 风险/反驳

**目标**：展示客观性，增加可信度

```
反方观点：[...]

我为什么仍然认为 [结论]：[...]
```

---

### 第 10 条 — 结论与行动建议

**目标**：给出明确结论，可操作的建议

```
总结：[一句话结论]

行动建议：
• [如果 A，则 X]
• [如果 B，则 Y]

⚠️ 以上仅为个人分析，不构成投资建议。
```

---

### 最后一条 — 互动引导（可选）

```
你怎么看这个逻辑？

有不同意见欢迎讨论 👇

如果觉得有用，转发给可能需要的朋友。
```

---

## Thread 写作要点

- 每条推文可以独立成立，不需要前后文也能看懂
- 避免悬念结尾（"下一条揭晓"），读者会直接跳过
- 数据一定要标注来源和时间
- 用数字编号（1/10, 2/10...）方便读者追踪进度
- 情绪化表达克制，数据说话
